Detects various issues in Dockerfiles that could lead to security vulnerabilities, package management inefficiencies, and bloated image sizes. Addressing these issues enhances the security, maintainability, and efficiency of Docker images.

  • Unsafe ARG in RUN: Using ARG variables in RUN commands can be overridden by users, leading to unintended behaviors or security risks.
  • Curl Bashing: Detects the use of curl or wget with pipes or redirections to execute scripts from untrusted sources, preventing the execution of potentially harmful scripts.
  • 'apt-get dist-upgrade' Used: Avoids using 'apt-get dist-upgrade' in Dockerfiles, as it performs major version upgrades which are inappropriate for Docker images.
  • 'apt-get' Missing '-y': Ensures 'apt-get' commands include the '-y' flag to avoid manual user input during package installations.
  • 'zypper clean' Missing: Ensures that cached package data is cleaned after using zypper to reduce image size.
  • RUN Package-Manager Update Alone: Ensures that 'RUN <package-manager> update' instructions are not used alone and are followed by install commands in the same RUN statement to prevent orphaned packages.
  • Missing dnf Clean: Ensures 'dnf clean all' is used after 'dnf install' commands to clean package caches and reduce image size.
  • 'yum clean all' Missing: Ensures 'yum clean all' is used after 'yum install' commands to clean package caches and reduce image size.
  • RUN cd ...: Encourages the use of WORKDIR instead of 'RUN cd ...' commands to change directories, improving readability and maintainability.
